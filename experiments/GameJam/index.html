Understanding JavaScript and coding in general

JavaScript is the code of the web, a guide to tell the browser what you want it to do and how you want it done. You can say that coding is an art form, because it is not something so intuitive as most other things. But that shouldn't discourage you because in order to understand coding, you must understand that it requires a different way of looking and thinking about things. It requires a more logical, more rigid way of thinking, computational thinking if you may, than just an assumption presumption of human thinking.

This means that the normal way of thinking through a problem is always a gray line, like the word gray itself. You can spell it gray or grey, it is accepted both ways and even pronounced the same. Which is fine, but computers don't understand that. Computers can only view the world in a black or white fashion. It is either true or false, 0 or 1. For computers they can only accept gray, because that is how it is programmed in and that is what they know to be true. This kind of thinking really stumps people when they program, understand how to think this way and you are one step closer to learning how to program!

Next thing you should also understand is that you never stop learning things about coding. Look at examples, dissect them, play around with them, experiment with different things here and there. If something doesn't work or is getting hard to do, try something else or look at it in a different angle. The more you play around with things the more you are learning about them and will slowly develop ideas on how to use them. And don't forget if you get stuck and can't get out, ask around! The internet has bundles of helpful forums and communities for programming in general. Or ask someone you know that understands coding to help you with the problem or help you understand the problem.

You can honestly start with any computer language when you first learn to code, but the easiest ones are BASIC, which is kind of self explanitory, and JavaScript. Which with JavaScript you can actually show to your friends, family, rivals, etc.

Functions

Right! Now to the nitty-gritty. So about 99% of your code will be encapsulized into functions, and if you have taken any kind of basic mathematics, you learn pretty early on what a function is. It looks something akin to this:

\[f ( x ) = x^2 + 3x - 5\]

Which in programming is extremely similar, it will look like this:

function f(x) { return (x*x + 3*x - 5); }

But unlike mathematics, where you would traditionally name those functions f(x), g(x), and h(x) respectively, you are able to name the functions whatever you like, but it is recommended to name them something descriptive. Like instead of naming it just f(x), you can name it "parabola", "createParabola", or something of that sort. Because there will be tens to thousands of functions, and you won't remember what f(x) or d(x) or t(x) would do. And not only that but the parameters that you put in doesn't have to be x, or even just one thing. You can put as much as you like, or nothing at all! The names of the parameters should also be descriptive to better understand how it will be used. Not to mention that they don't necessarily need to be a number either it could be any kind of data type you would like to be. [ We will get to data types in a moment ] Here's an example different kinds of functions:

function displayPerson(firstName, lastName, age) { alert("Hello "+lastName+", "+firstName+"! You are "+age+" years old!" ); // This would display 'Hello Gonzalez-Becerra, Paul! You are 20 years old!' if you called // the function from somewhere like so 'displayPerson("Paul", "Gonzalez-Becerra", 20);' }

// It is also a good idea to comment on top of the function a super short summary of what it does
function displayOkSign() { alert("You are good to go!"); }

Now from this you should start to get the general gist of how things are being done. In fact the alert() you see within the functions is a function itself. Which, if you haven't already played with it, displays a dialog box top-centered of the web browser of whatever object is within the parenthesis. Heck you can even put a function as a parameter, and it will work just dandy. Although when you do that you must either provide the name of the function, or have a function right there as represented below.

// This will take a function, which assumes it is an equation, and displays what y is when x = 5
function displayFunction(func) { alert(func(5)); }

// This is a function for the parabola [ x^2 - 3x + 5 ]
function createParabola(x) { return (x*x - 3*x + 5); }

// This will display '45'
displayFunction(createParabola);

// This will display '250'
displayFunction(function(x) { return (x*x*x + 5*x*x); });

One last thing, for JavaScript it is runned the second it is detected. Which means that if you have code at the top of your page that messes with something at the bottom of your page, it won't be able to since it hasn't loaded in yet. In order to counteract this, there are two things you can do:

  1. Put the code at the bottom of your page, so it will be the last thing you load. Which is a good idea, but that will usually get lost in your webpage code, which might be a pain to most, including me.
  2. Put your code at the top of your page, but load an event to call on your code when everything is up and ready, which will look like such:

window.addEventListener(function(args) { // Your code starts here });
Brief look at data types and automatic variables

I say a brief look at it because JavaScript supports data types but uses the automatic variable for everything. I mean EVERYTHING, it is pretty ridiculous and mind-boggling if you are not used to it. Essentially a data type tells the computer what the variable can do, what functions it has, what properties and variables it has inside it, the whole shebang! Which is great so you know exactly what you are getting and how to mess with thing accordingly. And automatic variables basically throw all that out the window! What automatic variables do is basically adjust to whatever you tell it to be. So if you want your variable, lets call it x, to represent a number, you assign it a number, like 4. Ok cool your variable is now 4, but what if you want to have it as a sentence for the age of your nephew. then just set that sucker to "4 years old" and it will adjust accordingly. Want a function on it or a variable within it? You can do that, but it is alittle bit weirder, because that goes into creating classes/objects. Example:

// Variables
var x = 4; // This is creating a variable and setting it to 4

// This is setting the variable to be 4 years old
// Note that x += " years old"; means the same thing as x = x+" years old";
x += " years old";
// Displays '4 years old'
alert(x);
