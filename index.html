For the COS SURGE People:
Here are a couple of guidelines to getting started with the SURGE project.
  1. Sign up for GitHub: There is a website where you can upload your code for a project and it will tie everything in together on the parts where you editted, no nasty overrides. Check it out at GitHub.com

  2. Understand how to use GitHub: Sounds simple, but it is quite tricky to understand at first. Don't know where to start? GitHub has a thing to help you, here, although it is still a little vague. It basically wants you to download a seperate program that it works in tangent with, Git. And don't worry everything is free! Here are some pointers to try and get the grasp of GitHub
    • Make a couple of projects and then delete them later
    • Try to upload and manage a couple of files, doesn't have to be the best or make sense, just try uploading a .txt somehow
    • Try to fork someone's work, and mess around with it. Doesn't destroy the original's work.
    • Go to the team leader's GitHub and fork one of his works, and try to improve it somehow. So we can get the feel of cooperating, without much cooperating.

  3. Download Visual Studio Express: Visual Studio Express is free, but the non-Express version is not. They are also on DreamSpark. Doesn't necessarily have to be visual studio, but the team leader prefers it. As long as you are able to host the website in a local host or something on a server, and not on a local file system, you should be good. Heck, GitHub provides a nice 'website' hosting for these kinds of things, here. Why? I certainly don't understand why, but some of the stuff we might be making may not work on a local file system.

  4. Understand Bootstrap: Bootstrap is a staple nowadays for any kind of website designer. It's sleek, it's nice, and it's beautiful, it makes the 90's look of a website look like to 2010's average phone app, without any real effort. It is just a bunch of CSS and JavaScript that can be easily downloaded here, and documentation here for CSS and JavaScript. Although we will most likely be using the CSS portion of it.

  5. "Study Up" on the current works of the Team Leader: There isn't much on the team leader's GitHub but there is a repository called " SurgeWebSite". Look at the code, try and understand what is being done, learn from it, eat with it, and sleep with it. The more you understand how the project is pieced together, the better. Although a large portion of that project using JSXGraph, which as of right now, it is still undecided for the project.
